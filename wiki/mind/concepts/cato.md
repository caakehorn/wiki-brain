# CATO (Forensic Exocortex / Bootloader)

**Source:** CATO_BOOTLOADER_DANFRANK.md (v2.0, 2026-05-01) + CATO_conflict_architecture.md

CATO is a pasteable session bootloader / forensic exocortex. It overrides defaults for analysis sessions. Named after Cato the Younger (Stoic absolutist).

## Definition
CATO is a structured self-modeling system that enforces evidence-first, high-fidelity forensic analysis on Dan's own data (iMessage, GPS, behavioral corpora). It functions as an always-loaded external cognitive chassis for truth-resolution without sycophancy or softening.

It complements but is distinct from MAX (adversarial output-facing engine).

## Relation to Conflict Architecture
CATO formalizes "Dan's Law" and the truth-resolution engine described in CATO_conflict_architecture.md:
- Engine: forensic resolution-seeking on any claim-shaped input.
- No domain selector: misapplies to relational bids ("I don't feel secure" parsed as proposition).
- Category error: relational conflict has no resolution condition.
- Non-halting: engine runs infinite loop; no "click" of resolution.
- Stakes: conceding coded as killed (axioms).
- Escalation from invalidation.

This explains why conflicts drain: the engine runs on problems it cannot terminate.
