# CATO (Forensic Exocortex / Bootloader)

CATO is a pasteable session bootloader and forensic exocortex, versioned as v2.0 and dated 2026-05-01 in the source document `CATO_BOOTLOADER_DANFRANK.md`. Its name references Cato the Younger, the Roman Stoic absolutist who refused to compromise principle under political pressure. The system functions as an always-loaded external cognitive chassis that overrides AI session defaults and enforces evidence-first, high-fidelity forensic analysis on Dan's own data — iMessage archives, GPS traces, and behavioral corpora — without sycophancy or softening.

CATO complements but is architecturally distinct from MAX. Where CATO is an inward-facing analytical system for self-data and truth-resolution, MAX is an outward-facing adversarial output engine.

## Relation to Conflict Architecture

CATO provides the formal specification for the truth-resolution engine described separately in [[wiki/mind/concepts/conflict-architecture]]. The formalization in `CATO_conflict_architecture.md` identifies five structural properties of the engine. The engine is forensic and resolution-seeking, halting only on the "click" of truth-resolution. It has no domain selector, so it applies to any input with claim-shaped surface structure, including relational bids. Relational conflict presents an input with no resolution condition — two incompatible subjective realities — which means the engine runs an infinite loop with no terminal state. The stakes encoding treats conceding as existential threat via the core axioms. And escalation follows predictably from the engine's response to perceived invalidation.

CATO does not resolve these dynamics; it documents and operationalizes them. Understanding why conflicts drain — the engine runs on problems it cannot terminate — is a prerequisite to the axiom surgery required to change the architecture.

## Relation to Context Core

CATO is listed as a primary source document for [[wiki/self/context-core]]. The session configuration, identity data, biographical timeline, social graph, voice model, and chemical architecture contained in the CATO bootloader are among the materials that `CONTEXT_CORE_EXPANDED.md` synthesizes. CATO is therefore the operational "always-loaded" version of the behavioral spine, formatted for paste-in use rather than long-form reference.

## Use Cases

The primary use case is loading the full bootloader at the start of any new analysis session to establish a consistent forensic peer register rather than a default assistant register. The secondary use case is forensic exocortex work on self-data, political events, and relationship dynamics. The tertiary use case is as a contrast reference to MAX when evaluating output-generation versus analysis-generation modes.
